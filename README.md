# MegaD-2561 Integration for Home Assistant

[![GitHub Release](https://img.shields.io/github/v/release/zmej904/megad)](https://github.com/zmej904/megad/releases)
[![HACS Default](https://img.shields.io/badge/HACS-Default-orange.svg)](https://github.com/hacs/integration)
[![GitHub License](https://img.shields.io/github/license/zmej904/megad)](LICENSE)

Интеграция для управления контроллерами **MegaD-2561** в Home Assistant.

Поддерживает реле, ШИМ, датчики (1-Wire, I2C, DHT), сенсорные кнопки, ПИД-регуляторы и многое другое.

---

## 📋 Возможности

- **Управление выходами**: реле (`switch`/`light`), ШИМ-регуляторы (`light` с поддержкой яркости)
- **Обработка входов**: бинарные сенсоры (режимы P, R, P&R), сенсорные кнопки (одинарное/двойное/длинное нажатие)
- **Датчики**: температура/влажность (DHT, 1-Wire, I2C: HTU21D, SHT31 и др.), освещённость (BH1750, MAX44009), CO2 (SCD4x, T67xx), давление (BMP180, MBx280), ток (INA226)
- **Расширители портов**: MCP230xx, PCA9685 через шину I2C
- **Термостаты**: встроенные ПИД-регуляторы, климат-устройства
- **Watchdog**: автоматический мониторинг доступности и восстановление связи с контроллером
- **Webhook**: приём данных от MegaD через HTTP-эндпоинт `/megad` (порт 8123)

## 📥 Установка

### Способ 1: HACS (рекомендуется)

1. Установите [HACS](https://hacs.xyz/) если ещё не сделали
2. Добавьте пользовательский репозиторий:
   - `HACS` → `Интеграции` → `⋮` → `Пользовательские репозитории`
   - URL: `https://github.com/zmej904/megad`
   - Категория: `Интеграция`
3. Нажмите `Установить` и выберите последнюю версию
4. **Полностью перезапустите Home Assistant**

### Способ 2: Ручная установка

1. Скачайте последний релиз из раздела [Releases](https://github.com/zmej904/megad/releases)
2. Распакуйте архив и скопируйте папку `megad` в директорию `custom_components` вашего Home Assistant
3. **Перезапустите Home Assistant**

## ⚙️ Настройка интеграции

1. **Добавьте устройство MegaD**:
   - `Настройки` → `Устройства и службы` → `+ Добавить интеграцию` → `MegaD-2561`

2. **Введите параметры подключения**:
   - **IP-адрес** контроллера (например, `192.168.1.14`)
   - **Пароль** (по умолчанию `sec`)

3. Интеграция автоматически считает конфигурацию с контроллера и создаст все необходимые сущности

## 📡 Настройка обратной связи (мгновенное обновление)

Для мгновенного обновления состояний в Home Assistant при изменениях на контроллере необходимо настроить отправку данных на сервер.

### Инструкция для MegaD:

1. Войдите в веб-интерфейс контроллера: `http://IP_вашего_MegaD/sec/`
2. Нажмите на гиперссылку **`Config`** в верхней части страницы
3. Перейдите на вкладку **`CF1`** (основные настройки сети и сервера)
4. Настройте следующие параметры:

| Параметр | Значение | Пояснение |
|----------|----------|-----------|
| **`SRV Type`** | `HTTP` | Выберите HTTP из выпадающего списка (доступно: HTTP или MQTT) |
| **`SRV`** | `IP_вашего_HA:8123` | Адрес и порт Home Assistant, например `192.168.1.100:8123` |

5. Нажмите кнопку **`Save`** / **`Сохранить`**

**Контроллер автоматически перезагрузится сам** — ничего дополнительно нажимать не нужно.

### Важно:

- **`SRV Type`** должен быть **`HTTP`** (не MQTT)
- **Порт 8123** — стандартный порт Home Assistant
- **IP-адрес** Home Assistant должен быть статическим (не меняться после перезагрузки)

### Проверка работы:

1. Выполните любое действие на контроллере (нажмите кнопку, переключите порт)
2. Откройте логи Home Assistant: `Настройки` → `Система` → `Логи`
3. Должна появиться запись:
4. 4. Сенсор `MegaD megad Feedback Status` покажет `active` или `ok`

### Если обратная связь не работает:

1. ✅ `SRV Type` = `HTTP` (не MQTT!)
2. ✅ `SRV` = `IP_вашего_HA:8123` (проверьте адрес и порт)
3. ✅ Порт 8123 должен быть доступен с контроллера
4. ✅ Временно отключите брандмауэр на сервере HA
5. ✅ Включите отладку в `configuration.yaml`:
```yaml
logger:
  logs:
    custom_components.megad.server: debug
